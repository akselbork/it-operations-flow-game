# IT Operations Flow Game

A systems thinking simulation for IT Operations, reliability, rework, hidden dependencies and cross-team flow.

## Purpose

The IT Operations Flow Game is designed to help IT teams experience how operational systems behave under limited capacity, delayed information, hidden dependencies and local optimization.

The game does not have a perfect solution.

The goal is not to complete all work.
The goal is to make system behavior visible.

Participants should experience how:

* limited capacity affects prioritization
* quick fixes can create rework
* root cause analysis competes with daily operations
* local decisions can create work for other teams
* delayed information changes decision quality
* poor communication increases Service Desk load
* user perception can differ from technical status

## Core idea

In IT Operations, teams often make rational decisions based on the information available to them.

But when the system contains delays, hidden dependencies, unclear ownership and limited feedback, those local decisions can still create instability across the organization.

The game is built to show that difference.

## Main roles

The first version of the game uses three technical teams and one perception layer:

* Network / DHCP
* IAM / DNS
* Server / Storage / Backup
* Service Desk

Service Desk does not solve root cause.
Service Desk shows how the organization experiences IT.

## Current status

Version 0.1 is an early concept.

The first goal is to build a simple playable paper-based prototype that can demonstrate:

* backlog growth
* rework
* recurring incidents
* delayed consequences
* hidden dependencies
* Service Desk pressure
* reduced trust
* improved outcomes when communication and learning capacity are added
